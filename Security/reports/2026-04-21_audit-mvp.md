## 1. Vulnérabilités CRITICAL 🔴

### CRIT-001: Absence Totale d'Authentification
**Sévérité**: CRITICAL  
**OWASP**: A01:2021 - Broken Access Control  
**Impact**: Accès non autorisé à toutes les données utilisateur

**Description**:
- Tous les endpoints sont publics et accessibles sans authentification
- UserID hardcodé dans les handlers: `00000000-0000-0000-0000-000000000001`
- Aucune validation de l'identité de l'utilisateur

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/http/handlers/collection_handler.go:85
userID := uuid.MustParse("00000000-0000-0000-0000-000000000001")

// backend/collection-management/internal/infrastructure/http/handlers/catalog_handler.go:48
userID := uuid.MustParse("00000000-0000-0000-0000-000000000001")

// backend/collection-management/internal/infrastructure/http/handlers/activity_handler.go:64
userID := uuid.MustParse("00000000-0000-0000-0000-000000000001")
```

**Preuve de Concept**:
```bash
# N'importe qui peut accéder aux données de "l'utilisateur fictif"
curl http://localhost:8080/api/v1/collections/summary
# → Retourne les données sans authentification
```

**Recommandation**:
- Implémenter JWT authentication avec middleware
- Extraire userID depuis le token JWT validé
- Rejeter toute requête sans token valide (sauf endpoints publics)

**Priorité**: P0 - Bloquant pour la production

### CRIT-002: Injection SQL dans CardRepository.GetCardsCatalog
**Sévérité**: CRITICAL  
**OWASP**: A03:2021 - Injection  
**Impact**: Exfiltration de données, modification/suppression de base de données

**Description**:
La construction dynamique de requêtes SQL avec `fmt.Sprintf` crée une vulnérabilité d'injection SQL, bien que partiellement atténuée par l'utilisation de paramètres positionnels.

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/postgres/card_repository.go:53-90
where := []string{}
if filter.Search != "" {
    where = append(where, fmt.Sprintf("(LOWER(c.name_fr) LIKE LOWER($%d) OR LOWER(c.name_en) LIKE LOWER($%d))", idx, idx+1))
    like := "%" + strings.ToLower(filter.Search) + "%"  // ❌ Manipulation de chaîne
    args = append(args, like, like)
}
// ...
countQuery := fmt.Sprintf(`SELECT COUNT(*) FROM cards c [...] WHERE [...] %s`, whereClause)
dataQuery := fmt.Sprintf(`SELECT [...] WHERE [...] %s [...] LIMIT $%d OFFSET $%d`, whereClause, idx, idx+1)
```

**Problème**:
- Bien que les valeurs utilisent des placeholders `$N`, la construction avec `fmt.Sprintf` introduit un risque
- La manipulation de `filter.Search` avec string concatenation est dangereuse
- Le LIKE pattern est construit manuellement au lieu d'utiliser les paramètres

**Preuve de Concept**:
```bash
# Tentative d'injection (actuellement bloquée par les placeholders, mais structure à risque)
curl "http://localhost:8080/api/v1/cards?search=%27%20OR%20%271%27=%271"
```

**Recommandation**:
Utiliser un Query Builder sécurisé ou refactoriser avec des paramètres stricts:

```go
// Approche sécurisée
const baseQuery = `SELECT COUNT(*) FROM cards c LEFT JOIN user_cards uc [...] WHERE c.collection_id IN (SELECT [...]) `
whereClauses := []string{}
if filter.Search != "" {
    whereClauses = append(whereClauses, "(LOWER(c.name_fr) LIKE LOWER($" + strconv.Itoa(idx) + ") OR LOWER(c.name_en) LIKE LOWER($" + strconv.Itoa(idx+1) + "))")
    args = append(args, "%"+filter.Search+"%", "%"+filter.Search+"%")
    idx += 2
}
finalQuery := baseQuery
if len(whereClauses) > 0 {
    finalQuery += " AND " + strings.Join(whereClauses, " AND ")
}
```

**Priorité**: P0 - Critique

### CRIT-003: CORS Mal Configuré (Développement)
**Sévérité**: CRITICAL (en production)  
**OWASP**: A05:2021 - Security Misconfiguration  
**Impact**: Vulnérabilités XSS cross-origin, attaques CSRF

**Description**:
La configuration CORS accepte dynamiquement les origins `localhost:3000` et `localhost:3001` sans validation stricte et expose tous les headers.

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/http/server.go:52-69
s.router.Use(func(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        origin := r.Header.Get("Origin")
        // ❌ Acceptation basée sur simple string matching
        if origin == "http://localhost:3000" || origin == "http://localhost:3001" {
            w.Header().Set("Access-Control-Allow-Origin", origin)
        }
        w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
        w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")
        // ❌ Pas de Access-Control-Allow-Credentials
        // ❌ Pas de validation de méthodes autorisées par endpoint
        if r.Method == "OPTIONS" {
            w.WriteHeader(http.StatusOK)
            return
        }
        next.ServeHTTP(w, r)
    })
})
```

**Problèmes**:
1. Aucune whitelist configurable (hardcodé dans le code)
2. Pas de protection CSRF
3. Manque `Access-Control-Max-Age` (cache preflight)
4. Configuration non adaptée à la production

**Recommandation**:
```go
// Configuration via variables d'environnement
allowedOrigins := strings.Split(os.Getenv("CORS_ALLOWED_ORIGINS"), ",")
corsMiddleware := cors.New(cors.Options{
    AllowedOrigins:   allowedOrigins,
    AllowedMethods:   []string{"GET", "POST", "PUT", "DELETE"},
    AllowedHeaders:   []string{"Content-Type", "Authorization"},
    AllowCredentials: true,
    MaxAge:           300,
})
s.router.Use(corsMiddleware.Handler)
```

**Priorité**: P0 - Bloquant pour la production

### CRIT-004: Credentials en Clair dans Docker Compose
**Sévérité**: CRITICAL  
**OWASP**: A02:2021 - Cryptographic Failures  
**Impact**: Exposition des credentials de base de données

**Description**:
Les credentials PostgreSQL sont hardcodés dans `docker-compose.yml` sans utilisation de secrets Docker.

**Localisation**:
```yaml
# backend/collection-management/docker-compose.yml:8-10
environment:
  POSTGRES_USER: collectoria      # ❌ Hardcodé
  POSTGRES_PASSWORD: collectoria  # ❌ Mot de passe faible hardcodé
  POSTGRES_DB: collection_management
```

**Problèmes**:
1. Credentials en clair dans le repository Git
2. Mot de passe faible (`collectoria`)
3. Pas d'utilisation de Docker secrets
4. Même credentials utilisés partout (dev/test/prod)

**Recommandation**:
```yaml
# Utiliser des variables d'environnement
environment:
  POSTGRES_USER: ${DB_USER}
  POSTGRES_PASSWORD: ${DB_PASSWORD}
  POSTGRES_DB: ${DB_NAME}

# Ou avec Docker secrets (production)
secrets:
  - db_password
environment:
  POSTGRES_PASSWORD_FILE: /run/secrets/db_password
```

**Priorité**: P0 - Critique

### CRIT-005: Pas de Rate Limiting
**Sévérité**: CRITICAL  
**OWASP**: A05:2021 - Security Misconfiguration  
**Impact**: DoS, brute force, scraping de données

**Description**:
Aucun rate limiting n'est implémenté sur les endpoints publics, permettant des attaques par déni de service.

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/http/server.go:44-49
func (s *Server) setupMiddleware() {
    s.router.Use(middleware.RequestID)
    s.router.Use(middleware.RealIP)
    s.router.Use(middleware.Logger)
    s.router.Use(middleware.Recoverer)
    s.router.Use(middleware.Timeout(60 * time.Second))
    // ❌ Pas de rate limiting
}
```

**Preuve de Concept**:
```bash
# Bombardement sans limite
for i in {1..10000}; do
  curl http://localhost:8080/api/v1/cards &
done
# → Le serveur accepte toutes les requêtes
```

**Recommandation**:
```go
import "github.com/didip/tollbooth/v7"
import "github.com/didip/tollbooth/v7/limiter"

// Limiter: 100 requêtes par minute par IP
rateLimiter := tollbooth.NewLimiter(100, &limiter.ExpirableOptions{
    DefaultExpirationTTL: time.Minute,
})
s.router.Use(tollbooth.LimitHandler(rateLimiter))
```

**Priorité**: P0 - Bloquant pour la production

## 2. Vulnérabilités HIGH 🟠

### HIGH-001: Pas de Validation d'Input sur les Query Parameters
**Sévérité**: HIGH  
**OWASP**: A03:2021 - Injection  
**Impact**: Crash application, comportement inattendu

**Description**:
Les paramètres de requête HTTP ne sont pas validés strictement, permettant des valeurs aberrantes.

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/http/handlers/catalog_handler.go:51-62
page := 1
limit := 50
if p := q.Get("page"); p != "" {
    if v, err := strconv.Atoi(p); err == nil && v > 0 {
        page = v  // ❌ Pas de limite supérieure
    }
}
if l := q.Get("limit"); l != "" {
    if v, err := strconv.Atoi(l); err == nil && v > 0 && v <= 200 {
        limit = v
    }
}
// ❌ filter.Search, filter.Series, etc. pas validés (longueur, caractères)
```

**Problèmes**:
1. `page` peut être un nombre gigantesque (ex: `page=999999999`)
2. `search` peut contenir des milliers de caractères
3. Pas de sanitization des inputs
4. Pas de limite sur les valeurs de filtres

**Recommandation**:
```go
func validatePagination(page, limit int) error {
    if page < 1 || page > 10000 { return errors.New("page must be 1-10000") }
    if limit < 1 || limit > 200 { return errors.New("limit must be 1-200") }
    return nil
}

func validateSearch(s string) error {
    if len(s) > 100 { return errors.New("search too long") }
    if strings.ContainsAny(s, "<>{}[]") { return errors.New("invalid characters") }
    return nil
}
```

**Priorité**: P1 - Haute

### HIGH-002: Erreurs Verboses Exposant des Détails Internes
**Sévérité**: HIGH  
**OWASP**: A05:2021 - Security Misconfiguration  
**Impact**: Information Disclosure, aide aux attaquants

**Description**:
Les erreurs retournent des messages génériques mais les logs peuvent exposer des détails sensibles.

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/http/handlers/collection_handler.go:89-91
if err != nil {
    h.logger.Error().Err(err).Msg("Failed to get collection summary")
    writeJSONError(w, h.logger, http.StatusInternalServerError, "INTERNAL_ERROR", "Failed to fetch collection summary")
    return
}
```

**Problèmes**:
1. Message d'erreur générique OK, mais logs en production exposent potentiellement des stack traces
2. Pas de distinction entre erreurs business et erreurs système
3. Logger configuré en mode développement (`zerolog.ConsoleWriter`)

**Recommandation**:
```go
// En production: JSON logger sans stack traces
if os.Getenv("ENV") == "production" {
    log.Logger = zerolog.New(os.Stdout).With().Timestamp().Logger()
} else {
    log.Logger = log.Output(zerolog.ConsoleWriter{Out: os.Stderr})
}

// Différencier les erreurs
switch err {
case ErrNotFound:
    writeJSONError(w, h.logger, http.StatusNotFound, "NOT_FOUND", "Resource not found")
case ErrValidation:
    writeJSONError(w, h.logger, http.StatusBadRequest, "VALIDATION_ERROR", err.Error())
default:
    h.logger.Error().Err(err).Msg("Internal error")  // Log détaillé
    writeJSONError(w, h.logger, http.StatusInternalServerError, "INTERNAL_ERROR", "An error occurred")  // Message générique
}
```

**Priorité**: P1 - Haute

### HIGH-003: Pas de Timeout sur les Requêtes SQL
**Sévérité**: HIGH  
**OWASP**: A04:2021 - Insecure Design  
**Impact**: Resource exhaustion, DoS

**Description**:
Les requêtes PostgreSQL n'ont pas de timeout explicite, pouvant bloquer indéfiniment.

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/postgres/collection_repository.go:33
err := r.db.SelectContext(ctx, &collections, query)
// ❌ Le context n'est pas configuré avec timeout
```

**Recommandation**:
```go
// Dans les handlers, créer un context avec timeout
ctx, cancel := context.WithTimeout(r.Context(), 5*time.Second)
defer cancel()

// Ou au niveau du serveur
s.router.Use(middleware.Timeout(30 * time.Second))  // Déjà en place

// Et au niveau DB
db.SetConnMaxLifetime(time.Minute * 3)
db.SetMaxOpenConns(25)
db.SetMaxIdleConns(5)
db.SetConnMaxIdleTime(time.Minute * 5)
```

**Priorité**: P1 - Haute

## 3. Vulnérabilités MEDIUM 🟡

### MED-001: Pas de Headers de Sécurité HTTP
**Sévérité**: MEDIUM  
**OWASP**: A05:2021 - Security Misconfiguration

**Headers Manquants**:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Strict-Transport-Security` (HSTS)
- `Content-Security-Policy`

**Recommandation**:
```go
func securityHeadersMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("X-Content-Type-Options", "nosniff")
        w.Header().Set("X-Frame-Options", "DENY")
        w.Header().Set("X-XSS-Protection", "1; mode=block")
        w.Header().Set("Referrer-Policy", "strict-origin-when-cross-origin")
        if r.TLS != nil {
            w.Header().Set("Strict-Transport-Security", "max-age=31536000; includeSubDomains")
        }
        next.ServeHTTP(w, r)
    })
}
```

**Priorité**: P2 - Moyenne

### MED-003: Absence de Health Check Détaillé
**Sévérité**: MEDIUM  
**Impact**: Difficile de détecter les problèmes de connectivité DB

**Localisation**:
```go
// backend/collection-management/internal/infrastructure/http/server.go:76-79
r.Get("/health", func(w http.ResponseWriter, r *http.Request) {
    w.WriteHeader(http.StatusOK)
    w.Write([]byte(`{"status":"ok"}`))
})
// ❌ Ne vérifie pas la connexion DB
```

**Recommandation**:
```go
r.Get("/health", func(w http.ResponseWriter, r *http.Request) {
    ctx, cancel := context.WithTimeout(r.Context(), 2*time.Second)
    defer cancel()
    if err := db.PingContext(ctx); err != nil {
        w.WriteHeader(http.StatusServiceUnavailable)
        json.NewEncoder(w).Encode(map[string]string{"status": "unhealthy", "error": "database unreachable"})
        return
    }
    w.WriteHeader(http.StatusOK)
    json.NewEncoder(w).Encode(map[string]string{"status": "healthy"})
})
```

**Priorité**: P2 - Moyenne

### MED-006: Dockerfile Non Optimisé pour la Sécurité
**Sévérité**: MEDIUM  
**OWASP**: A05:2021 - Security Misconfiguration

**Localisation**:
```dockerfile
# backend/collection-management/Dockerfile:17-20
FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/  # ❌ Exécution en tant que root
COPY --from=builder /app/main .
CMD ["./main"]
```

**Problèmes**:
1. Exécution en tant que `root` (UID 0)
2. Pas de user non-privilégié
3. Base image `alpine:latest` (pas de version fixée)

**Recommandation**:
```dockerfile
FROM alpine:3.19
RUN apk --no-cache add ca-certificates && \
    addgroup -g 1000 collectoria && \
    adduser -D -u 1000 -G collectoria collectoria
WORKDIR /app
COPY --from=builder --chown=collectoria:collectoria /app/main .
USER collectoria
EXPOSE 8080
CMD ["./main"]
```

**Priorité**: P2 - Moyenne
