SECTION 1 : CONTEXTE DU PROJET
═══════════════════════════════════════════════════════════════════════════════

### Identité de l'Application
Collectoria est une application de gestion de collections personnelles pour collectionneurs de cartes à jouer, livres, et jeux de rôle. L'utilisateur peut suivre sa progression, consulter ses collections, et explorer ses items.

### Creative North Star : "The Digital Curator"
L'application n'est PAS une base de données utilitaire, c'est une **galerie d'exposition muséale haut de gamme**. Chaque collection est une exposition de qualité.

**Objectif émotionnel** : Fournir un sentiment d'**autorité tranquille** à travers :
- Espace de respiration (whitespace généreux, même sur mobile)
- Absence de lignes structurelles rigides (No-Line Rule)
- Profondeur tonale (Tonal Layering)
- Typographie haute contraste (Dual-Type : Manrope + Inter)
- Focus sur les visuels de la collection (images hero)

### CLARIFICATION IMPORTANTE
Le "hero progress" (section de statistiques globales en haut de page) a été **délibérément supprimé** sur le web. Ce n'est PAS un gap à combler, mais un **choix de design validé** pour l'expérience utilisateur. Le focus est mis directement sur les collections individuelles.

SECTION 3 : DESIGN SYSTEM "THE DIGITAL CURATOR"
═══════════════════════════════════════════════════════════════════════════════

### Règle Fondamentale : No-Line Rule 🚫

**INTERDIT** : Utiliser des bordures 1px solides pour sectionner le contenu.

**À LA PLACE** : Définir les frontières par changements de fond (Tonal Layering).

### Palette de Couleurs

**Surfaces (Tonal Layering)** :
- Background principal : #f8f9fa (surface)
- Cards collection : #ffffff (surface-container-lowest)
- Bottom navigation : #ffffff (surface-container-lowest)
- Sections intermédiaires (si présentes) : #f3f4f5 (surface-container-low)
- Shadow bottom nav : 0 -4px 16px rgba(25, 28, 29, 0.06) (vers le haut)

**Primary (Violet signature)** :
- Gradient : #667eea → #764ba2 (angle 135deg ou 90deg selon contexte)
- Usage : CTAs, progress bars, états actifs bottom nav
- Utilisation avec PARCIMONIE (ne pas surcharger)

**Text** :
- Primary : #191c1d (on-surface)
- Secondary : #44474a (on-surface-variant)
- On primary : #ffffff

### Typography (Dual-Type System)

**Manrope** (Headlines, autorité, premium) :
- Titres de collections : 1.25rem (20px), weight 700
- Headlines de sections : 1.125rem (18px), weight 700

**Inter** (Body, métadonnées, navigation) :
- Métadonnées, descriptions : 0.875rem (14px), weight 400
- Labels bottom nav : 0.625rem (10px), weight 400/600 (inactif/actif)

### Principes Visuels

1. **No-Line Rule** : Aucune bordure 1px solide. Définir les frontières par changements de fond (Tonal Layering).
2. **Tonal Layering** : Cards blanches (#ffffff) sur background gris clair (#f8f9fa) = "soft lift" architectural.
3. **Border Radius** : lg (16px) minimum pour cards, bottom nav, sections.
4. **Spacing** : Généreux mais adapté mobile (12-16px gaps, 16px padding).
5. **Gradient** : Avec parcimonie (progress bars, CTAs, onglet actif).
6. **Whitespace** : Le whitespace est un élément de design, pas un accident. Si ça ne rentre pas avec respiration, repenser le layout.
