═══════════════════════════════════════════════════════════════════════════════
                              DATA FLOW EXAMPLE
═══════════════════════════════════════════════════════════════════════════════

1. User opens homepage
   └─> Next.js SSR prefetches data

2. Frontend makes 4 parallel API calls:
   ├─> GET /api/v1/collections/summary
   ├─> GET /api/v1/collections
   ├─> GET /api/v1/activities/recent
   └─> GET /api/v1/statistics/growth

3. API Gateway validates JWT, checks Redis cache
   ├─> Cache HIT → Return cached response
   └─> Cache MISS → Route to microservice

4. Microservices query PostgreSQL
   ├─> Collection Service: Aggregate user_cards
   └─> Statistics Service: Query activities, growth_snapshots

5. Microservices return data + store in Redis

6. Frontend receives responses
   └─> React Query caches data
       └─> Progressive rendering of components

7. User adds a new card
   └─> POST /api/v1/cards
       └─> Collection Service processes
           ├─> Update PostgreSQL
           ├─> Publish CardAdded event to Kafka
           └─> Invalidate Redis cache
               └─> Statistics Service consumes event
                   └─> Creates Activity record
                       └─> Frontend re-fetches data
