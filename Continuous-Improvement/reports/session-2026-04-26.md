## 2. Incidents et Leçons

### Incident A — JWT_SECRET manquant au démarrage

**Description**  
Lors du démarrage de l'environnement en début de session, le backend Go a refusé de démarrer. Cause : `JWT_SECRET` non défini dans la commande d'export des variables d'environnement du workflow de démarrage d'Alfred (`CLAUDE.md`).

**Cause Racine**  
Le workflow de démarrage de session dans `CLAUDE.md` (section "Démarrage de Session de Travail") liste les variables d'environnement à exporter pour le backend :
```
export DB_HOST=localhost DB_PORT=5432 DB_USER=collectoria DB_PASSWORD=collectoria DB_NAME=collection_management SERVER_PORT=8080
```
`JWT_SECRET`, `JWT_EXPIRATION_HOURS` et `JWT_ISSUER` sont absents de cette liste — alors que le backend les exige au démarrage (le code `config.go` retourne une erreur fatale si `JWT_SECRET` est absent ou < 32 caractères).

La procédure correcte existe dans `DevOps/restart-procedures.md` (méthode manuelle, ligne 79) mais n'est pas référencée dans le workflow automatique d'Alfred.

**Impact**  
Blocage immédiat au démarrage. Perte de temps en début de session (diagnostic, correction manuelle). Ce problème se reproduira à chaque nouvelle session si non corrigé.

**Recommandation**  
Mettre à jour la commande d'export dans `CLAUDE.md` (Alfred) ET dans `DevOps/CLAUDE.md` pour inclure les trois variables JWT. Correction appliquée dans ce rapport (voir section 3).

### Incident B — BookConfirmModal cassée (variables CSS inexistantes)

**Description**  
Le composant `BookConfirmModal.tsx` utilisait des variables CSS qui n'existent pas dans le design system défini dans `globals.css`. L'affichage était cassé (styles non appliqués, fallback sur valeurs CSS par défaut du navigateur).

**Cause Racine**  
L'agent Frontend (ou l'auteur initial du composant) a créé des variables CSS "de mémoire" sans vérifier leur existence dans `globals.css`. Les variables inventées n'étaient référencées nulle part dans le design system réel.

Variables réellement définies dans `globals.css` :
```
--surface, --surface-container-lowest, --surface-container-low,
--surface-container-high, --surface-container-highest,
--primary, --primary-container, --on-primary,
--on-surface, --on-surface-variant,
--font-editorial, --font-utility,
--radius-sm/md/lg/xl, --spacing-xs à --spacing-3xl
```

**Variable orpheline encore présente** : `--outline-variant` est utilisée dans `TopNav.tsx` (ligne 115 : `border: '1px solid var(--outline-variant)'`) mais n'est pas définie dans `globals.css`. Cela provoque un border invisible (fallback sur aucune valeur). Ce composant est en production active et présente donc un rendu dégradé silencieux.

**Impact**  
Rendu visuel incorrect sur BookConfirmModal (corrigé en session). Rendu dégradé silencieux sur le bouton "Se déconnecter" dans TopNav (border invisible).

**Recommandation**  
1. Formaliser une règle dans `Frontend/CLAUDE.md` : toujours consulter `globals.css` avant d'utiliser une variable CSS — jamais de variable inventée. Correction appliquée dans ce rapport.
2. Corriger `TopNav.tsx` ligne 115 : remplacer `var(--outline-variant)` par une valeur du design system (suggestion : `var(--surface-container-highest)` ou `rgba(25, 28, 29, 0.12)`).

### R4 — À PLANIFIER : Audit complet des variables CSS orphelines

Effectuer une passe de vérification sur tous les composants `.tsx` pour détecter toute autre variable CSS utilisée mais non définie dans `globals.css`.

**Commande de détection** :
```bash
# Extraire toutes les variables utilisées dans les composants
grep -roh "var(--[a-zA-Z0-9-]*)" frontend/src/components/ frontend/src/app/ \
  | grep -v "globals.css" | grep -oP "var\(--[a-zA-Z0-9-]+\)" | sort | uniq \
  > /tmp/vars_used.txt

# Extraire toutes les variables définies dans globals.css
grep -oh "\-\-[a-zA-Z0-9-]*:" frontend/src/app/globals.css \
  | tr -d ':' | sort | uniq \
  > /tmp/vars_defined.txt

# Variables utilisées mais non définies
comm -23 /tmp/vars_used.txt /tmp/vars_defined.txt
```

**Résultat connu** : `--outline-variant` est la seule variable orpheline identifiée à ce jour.
